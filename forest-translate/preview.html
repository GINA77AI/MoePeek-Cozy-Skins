🎨 All Skins at a Glance
🏗️ Architecture

A single Theme.swift file defines every visual parameter. All UI components read from ThemeManager.shared.current. Adding a new skin = adding one static struct definition. Zero changes to business logic.

🎨 Theme.swift

Core definitions: Theme struct with ~40 parameters per skin, ThemeManager singleton, UserDefaults persistence, 5 built-in skins.

🪟 PopupPanel + PopupView

Background gradient, shadow radius, corner radius — all driven by Theme. Falls back gracefully if no custom theme is set.

🃏 ProviderResultCard

Card tint, border, text hierarchy (primary/secondary/provider/hint), status indicators (waiting/translating/success/error).

🔘 TriggerIconController

The floating trigger button drawn via Core Graphics — base color, glow, border, icon color all from Theme's NSColor properties.
